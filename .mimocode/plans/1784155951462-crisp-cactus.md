# Fix: duplicate `__cmd__get_connected_devices` symbol

## Root Cause

The source code in `src-tauri/src/lib.rs` is correct — there is exactly one
`#[tauri::command]` annotation on `get_connected_devices` (line 68) and one
entry in `generate_handler![]` (line 157). No duplicate in the source.

The real cause is **two stale fingerprint directories** in the incremental
build cache that each contain a compiled copy of the `tauri::command` macro
expansion for this function:

- `target/debug/.fingerprint/hotspot-proxy-app-3e4c3d49bcbc63c5/`
- `target/debug/.fingerprint/hotspot-proxy-app-4c890768a0091b8d/`

Both cached artifacts independently define `__cmd__get_connected_devices` and
`__tauri_command_name_get_connected_devices` in the macro namespace, causing
E0255 when the linker sees them together. This happens after a crate hash
changes (e.g. dependency version bump, Cargo.toml edit, or Tauri update) while
old objects remain.

## Fix

Run `cargo clean` in `src-tauri/` to wipe the entire `target/` directory and
force a full, clean recompile:

```bash
cd src-tauri && cargo clean
```

Then restart the dev server normally:

```bash
npm run tauri dev
```

**No source file changes required.**

## Verification

After `cargo clean`, the `target/debug/.fingerprint/` directory will be empty.
`npm run tauri dev` should compile without E0255 errors.
